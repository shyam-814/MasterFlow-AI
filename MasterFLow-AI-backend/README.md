# Backend Setup Guide

This backend application uses Express.js with TypeScript, Redis for caching/queues, and PostgreSQL with Prisma ORM.

## Prerequisites

- Node.js (v18 or higher)
- Docker (for Redis and PostgreSQL)
- npm or yarn package manager

## Quick Start

### 1. Install Dependencies

```bash
cd backend
npm install
```

### 2. Setup Environment Variables

Copy the example environment file and configure it:

```bash
cp .env.example .env
```

Update the `.env` file with your configuration:

```env
# Redis Configuration
REDIS_URL=redis://localhost:6379
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# Application settings
NODE_ENV=development
BACKEND_PORT=3000

# Database (will be generated by Prisma)
DATABASE_URL="your-database-url-here"
```

### 3. Start Redis Server

Using Docker (recommended):

```bash
# Start Redis container
docker run -d --name redis-flows -p 6379:6379 redis:alpine

# Verify Redis is running
docker ps
```

Alternative - Using local Redis installation:

```bash
# Install Redis (macOS)
brew install redis
redis-server

# Install Redis (Ubuntu/Debian)
sudo apt-get install redis-server
sudo systemctl start redis-server

# Install Redis (Windows)
# Download from https://github.com/MicrosoftArchive/redis/releases
```

### 4. Setup PostgreSQL Database

Start a local PostgreSQL database using Prisma:

```bash
# Start Prisma's local PostgreSQL server
npx prisma dev

# This will:
# 1. Start a local PostgreSQL instance
# 2. Create a database
# 3. Update your .env file with DATABASE_URL
# 4. Generate Prisma client
```

If you prefer using Docker for PostgreSQL:

```bash
# Start PostgreSQL container
docker run -d \
  --name postgres-flows \
  -e POSTGRES_PASSWORD=password \
  -e POSTGRES_DB=flows_db \
  -p 5432:5432 \
  postgres:15

# Update your .env file
DATABASE_URL="postgresql://postgres:password@localhost:5432/flows_db"
```

### 5. Generate Prisma Client

```bash
# Generate Prisma client (if not done automatically)
npx prisma generate

# Push database schema (if you have migrations)
npx prisma db push
```

### 6. Start the Backend Server

```bash
# Development mode with hot reload
npm run dev

# Or if you don't have a dev script, use:
npx ts-node-dev src/server.ts
```

## Development Workflow

### Adding Database Models

1. Update [`prisma/schema.prisma`](prisma/schema.prisma):

```prisma
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  name      String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

2. Push changes to database:

```bash
npx prisma db push
```

3. Regenerate Prisma client:

```bash
npx prisma generate
```

### Using Redis Cache and Queue

The Redis client is available via the [`redisClient`](src/redis/redisClient.ts) singleton:

```typescript
import { redisClient } from './redis/redisClient';

// Cache example
await redisClient.cacheSet('user:123', userData, { ttl: 3600 });
const user = await redisClient.cacheGet('user:123');

// Queue example
await redisClient.queuePush('tasks', { type: 'email', data: emailData });
const task = await redisClient.queuePop('tasks');
```

See [`src/redis/example.ts`](src/redis/example.ts) for comprehensive usage examples.

## Troubleshooting

### Redis Connection Issues

```bash
# Check Redis is running
docker ps | grep redis

# Check Redis logs
docker logs redis-flows

# Test Redis connection
redis-cli -h localhost -p 6379 ping
```

### Database Connection Issues

```bash
# Check PostgreSQL is running
docker ps | grep postgres

# Check database connection
npx prisma db push

# Reset database if needed
npx prisma db push --force-reset
```

### Common Issues

1. **Port already in use**: Change `BACKEND_PORT` in `.env`
2. **Redis connection refused**: Ensure Redis is running on correct port
3. **Database connection failed**: Verify `DATABASE_URL` in `.env`
4. **Prisma client not found**: Run `npx prisma generate`
